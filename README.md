# braille_cpp — 한글 점자 → 한글 텍스트 변환기 (C++17 포팅)

원본 파이썬 `conv()` 함수와 점자 사전들을 C++17로 그대로 포팅한 것.
브레이스 스타일은 요청대로 Allman(중괄호를 자기 줄에 위/아래로) 사용.

## 구조

```
include/
  utf8_util.hpp      UTF-8 <-> UTF-32 변환 (점자/한글은 전부 BMP라 1~3바이트면 충분)
  braille_maps.hpp    모든 점자 사전 선언
  jamo_compose.hpp    join_jamos() 자모 결합 함수 선언
  braille_conv.hpp    conv() 변환 함수 선언 (공개 API는 이거 하나)
src/
  braille_maps.cpp    사전 정의 (파이썬 원본에서 자동 생성, 손 전사 아님)
  jamo_compose.cpp    join_jamos() 구현
  braille_conv.cpp    conv() 본체
test/
  main.cpp            CLI 데모: ./braille_conv "⠈⠣"
```

## 빌드

```
g++ -std=c++17 -finput-charset=UTF-8 -fexec-charset=UTF-8 \
    -I include src/*.cpp test/main.cpp -o braille_conv
./braille_conv "⠈⠣"
```

소스 안에 점자/한글 유니코드 문자를 리터럴로 직접 박아 넣었기 때문에
`-finput-charset=UTF-8` (소스가 UTF-8 인코딩이라는 전제) 가 필요하다.
대부분의 리눅스 환경에서는 기본값이라 생략해도 되지만, 명시하는 게 안전하다.

## 원본과 다른 점 (전부 의도적, 동작은 동일하게 유지)

1. **`join_jamos` 구현체**
   원본은 외부 라이브러리(`jamotools.join_jamos`)를 그냥 가져다 썼고, 그
   소스는 제공되지 않았다. PyPI에서 실제 패키지를 찾아 설치한 뒤, 무작위
   생성 314개 케이스(겹받침, 모음 연쇄, 모호한 받침/초성 경계 등 포함)로
   `jamotools.join_jamos` 결과와 내가 직접 구현한 C++ `join_jamos`를
   1:1 비교해서 전부 일치하는 것을 확인했다 (diff 0).

2. **인덱스 범위 밖 접근에 대한 안전장치**
   파이썬은 `Str[i-1]`이 `i==0`일 때 뒤에서부터 wrap하고(이건 그대로
   재현했다), 진짜 범위를 넘으면 `IndexError`로 죽는다. 원본은 문자열
   앞뒤에 점자 빈칸 3개를 패딩해서 보통 이 경우에 도달하지 않는다.
   C++의 `operator[]`는 범위를 넘으면 UB이므로, 범위 밖 접근 시
   `U'\0'`(센티넬)을 반환하는 `Ch()` 헬퍼를 추가했다. 정상 입력에서는
   동작에 차이가 없고, 비정상/잘린 입력에서 크래시 대신 안전하게
   끝나도록 하는 방어 코드다.

3. **`⠸⠎` 분기의 원본 버그를 "고치지 않고" 그대로 포팅**
   `Str[i-1] == '⠠'`(쌍자음 뒤)일 때 원본은
   `Type_1_double_dict.get(Str[i+1], "")` 를 호출하는데, 여기서
   `Str[i+1]`은 단일 문자 `'⠎'` 뿐이다. 그런데 `Type_1_double_dict`에서
   `"껏"`은 2글자 키 `"⠸⠎"`에 매달려 있어서 단일 키 `'⠎'`로는 절대
   찾아지지 않는다. 즉 원본도 이 경로에서는 항상 빈 문자열만 붙는다
   (의도는 "껏"이었을 가능성이 높지만 실제로는 그렇게 동작하지 않음).
   `conv("⠠⠈⠣⠌")`(쌍자음 분기, 별개 케이스) 등으로 곁가지를 만져보고
   `conv("⠠⠸⠎")` 를 원본 파이썬(실제 `jamotools` 연결)으로 직접 돌려서
   `"⠀⠀⠀ㅅ⠀⠀⠀"`(즉 "것"도 "껏"도 안 붙음)가 나오는 걸 확인한 뒤
   C++도 동일하게 맞췄다. 임의로 "껏"을 채워 넣지 않았다.

## 검증한 것 / 안 한 것

- `pip install jamotools` 로 원본이 실제로 의존했을 라이브러리를 찾아
  설치하고, 사전 정의 + `conv()` 전체를 그대로 살린 파이썬 레퍼런스를
  만들어 **무작위 퍼즈 케이스 550개**와 **수동으로 짚은 분기별 케이스
  9개**를 C++ 빌드와 1:1 비교했다 — 전부 일치 (diff 0).
- 다만 이건 "원본 파이썬 코드와 동일하게 동작하는가"를 검증한 것이지,
  "이 변환기가 실제 한국어 점자 표기 규칙상 의미적으로 맞는 출력을
  내는가"까지 검증한 건 아니다. 후자는 실제 한국어 점자 변환이 필요한
  실문서/실전 데이터로 직접 대조해 보는 걸 권한다.
